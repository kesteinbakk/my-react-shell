| B003 | nested-popper-dismisses-dialog | awaiting-user-confirmation | — | **Filed 2026-06-30 (reported from offansk-ev Invite-member modal).** A `Select` (or any Radix popper) **inside** a `Dialog`/`Sheet`/`ConfirmDialog`: open the popper, then click inside the surface but outside the popper, and the **whole surface closes** — losing the user's in-progress data. Ruled out the `link:` Radix-dedup hazard by instrumentation (unified single Radix context). Root cause: while the popper owns the layer Radix forces the dialog content to `pointer-events: none`; the click falls through, Radix closes the popper, then **re-evaluates the same gesture** as an outside-interaction for the dialog and dismisses it — fired **deferred**, *after* the popper unmounts, so a "is a popper open now?" check is too late. **Fix (shell-side):** `src/components/useDialogDismissGuard.ts` — a shared hook that records (via a capture-phase `pointerdown` listener) whether a popper was open at **gesture start**, and `preventDefault()`s the surface's `onPointerDownOutside`/`onInteractOutside` for that gesture. Wired into `Dialog`, `Sheet`, `ConfirmDialog`. Verified in `my-react-shell-demo` (bug gesture survives; option-select / backdrop / Esc all still correct). | `src/components/useDialogDismissGuard.ts` (new), `src/components/{Dialog,Sheet,ConfirmDialog}.tsx`; demo `OverlaysDialogsPage.tsx`. |
| B002 | pageheader-unstable-thunk-loop | awaiting-user-confirmation | — | **Filed 2026-06-20 (dogfood from evaluering T029).** `usePageHeader` (the hook that replaced `<ShellPageHeader>` via T016/T017) **infinite-loops** ("Maximum update depth exceeded", crashes to the consumer's error boundary) when a consumer passes an **unstable option** — a fresh `title` / `tabs` thunk, a new `search` object, or an un-memoized `actions` array each render. The hook's **content effect** pushed `handle.update(spec)` on every option-identity change → `setState` on AppShell → band + consumer re-render → new identities → push again → loop (surfacing at the breadcrumb leaf's Radix Popper anchor). **Fixed (shell-side):** the content effect now guards the push behind a resolved-**value** comparison (`specsEquivalent` vs a `lastPushedRef`) — string thunks compared by resolved value, `actions` by a scalar-prop signature, `tabs`/`search` structurally — so unstable identities no longer push each render while a genuine value change still updates the band. Deepest-mounted-wins + in-place update preserved; docstring rewritten. **Related Radix-dedup note already covered** in `docs/guides/distribution-model.md`. Verified in `my-react-shell-demo` via live preview (pathological inline options → zero loop; a value change updates the leaf crumb live). | `src/app-shell/usePageHeader.ts` (content-effect value guard + docstring). |
